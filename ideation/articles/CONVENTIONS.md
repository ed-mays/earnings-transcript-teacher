# Article Conventions

## Folder Structure

One folder per track under `ideation/articles/`:

```
ideation/articles/
├── CONVENTIONS.md               # this file
├── publishing-sequence.proposed.md
├── diary/
├── building-with-ai/
├── llms-in-production/
└── non-coders-playbook/
```

## Tracks

| Folder | Track | Audience |
|--------|-------|----------|
| `diary/` | Diary | Cross-audience narrative |
| `building-with-ai/` | Building With AI | Developers |
| `llms-in-production/` | LLMs in Production | Applied AI engineers |
| `non-coders-playbook/` | Non-Coder's Playbook | PMs and founders |

## File Naming

```
[3-digit sequence]-[article-title-in-kebab-case].md
```

The sequence number reflects **publication order** across all tracks (not per-track order), matching the sequence defined in `publishing-sequence.proposed.md`.

### Examples

```
diary/001-why-i-built-an-earnings-transcript-parser-to-learn-ai.md
building-with-ai/002-the-claude-md-pattern-teaching-an-ai-your-codebase-conventions.md
non-coders-playbook/003-whats-actually-possible-when-you-build-with-ai-as-a-non-engineer.md
building-with-ai/004-ai-memory-systems-across-sessions-why-context-is-everything.md
```
